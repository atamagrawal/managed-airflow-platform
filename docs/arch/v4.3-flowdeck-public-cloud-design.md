## 1. Executive Summary

### 1.1 Purpose

This document defines the target architecture for AirflowCloud’s public cloud deployment on AWS and the control model that must remain stable across future hybrid and private deployment modes.

### 1.2 Architecture Goals

- Separate tenant intent, orchestration, and runtime execution.
- Keep the platform control surface centralized, auditable, and secure.
- Support deployment portability without changing the Control Plane contract.
- Isolate each customer deployment so failures do not spill across tenants.
- Provide consistent health, audit, and operational visibility across all layers.

### 1.3 Layer Model

| Layer | Responsibility | Location |
|---|---|---|
| **Control Plane** | User identity, organization/workspace state, billing, policies, audit, APIs, and deployment intent | AirflowCloud AWS |
| **Data Plane** | Deployment orchestration, reconciliation, command routing, health reporting, and secure communication with the Control Plane | AirflowCloud VPC or customer VPC |
| **Runtime Plane** | Airflow runtime resources, including webserver, scheduler, workers, triggerer, metadata DB, broker, and DAG storage | Same environment as the Data Plane for the selected mode |

### 1.4 Deployment Modes

| Mode | Data Plane | Runtime Plane | Ownership |
|---|---|---|---|
| **Public Cloud** | Managed by AirflowCloud | Managed by AirflowCloud | AirflowCloud operates both orchestration and runtime |
| **Hybrid** | Deployed into customer VPC | Deployed into customer VPC | AirflowCloud manages the Control Plane; customer-hosted planes connect outbound |
| **Private** | Deployed into customer VPC | Deployed into customer VPC | Customer operates the environment with the same platform contract |

### 1.5 Communication and Monitoring

The Control Plane persists desired state and sends orchestration commands to the correct Data Plane. The Data Plane maintains secure outbound communication, applies changes to the runtime environment, and reports health and status back. The same Data Plane also monitors multiple customer deployments independently and reconciles actual state against desired state on a continuous basis.

```yaml
  Type: JWT (RS256)

    - org_id: organization_id
    - roles: [role_assignments]
    - scopes: [permitted_scopes]
    - exp: expiration_time
```

```yaml
  States:
    - PENDING: Request received, awaiting processing
```

### 3.5 Control Plane and Data Plane Communication

The Control Plane communicates with each Data Plane through a secure, versioned, and idempotent service contract. The Control Plane owns desired state; the Data Plane owns orchestration and runtime reconciliation. In customer-hosted environments, the Data Plane initiates the connection to the Control Plane and maintains it for commands, acknowledgements, and status reporting.

#### Communication Patterns

| Direction | Use | Examples |
|---|---|---|
| Control Plane -> Data Plane | Orchestration intent | Create deployment, update config, scale workers, delete deployment |
| Data Plane -> Control Plane | Runtime feedback | Heartbeat, health, status, metrics summary, drift alert |
| Control Plane -> Data Plane | Reconciliation and retry | Status poll, command retry, config refresh |

#### Reliability Rules

- Commands must include a unique command ID and deployment ID.
- The Data Plane must treat commands as at-least-once deliveries.
- Status updates must be monotonic when possible and explicitly report failures when recovery is not possible.
- The Control Plane must tolerate delayed, duplicated, or out-of-order responses without corrupting deployment state.
- After a disconnect, the Data Plane must resume synchronization automatically.


## 4. Data Plane Design

### 4.1 Overview

The Data Plane is the **infrastructure orchestration layer** that manages Airflow deployments. It receives commands from the Control Plane and provisions/manages resources in the Runtime Plane. This is the **portable layer** that can run in AirflowCloud's VPC (Public Cloud) or customer's VPC (Hybrid/Private).

#### Agent Communication Protocol

```protobuf
// dataplane.proto

service DataPlaneService {
  // Bidirectional stream for commands and status
  rpc Connect(stream AgentMessage) returns (stream ControlMessage);
}

message AgentMessage {
  string agent_id = 1;
  string message_id = 2;
  
  oneof payload {
    Heartbeat heartbeat = 3;
    CommandResponse command_response = 4;
    StatusUpdate status_update = 5;
    MetricsReport metrics_report = 6;
  }
}

message ControlMessage {
  string message_id = 1;
  
  oneof payload {
    HeartbeatAck heartbeat_ack = 2;
    DeploymentCommand deployment_command = 3;
```

#### Deployment Controller Code Structure

```go
// pkg/controller/deployment/controller.go

package deployment

import (
    "context"
    "fmt"
    
    "github.com/airflowcloud/dataplane/pkg/terraform"
    "github.com/airflowcloud/dataplane/pkg/helm"
    "github.com/airflowcloud/dataplane/pkg/health"
    pb "github.com/airflowcloud/dataplane/proto"
)

type Controller struct {
    terraformRunner *terraform.Runner
    helmManager     *helm.Manager
    healthChecker   *health.Checker
    stateManager    *StateManager
    statusReporter  StatusReporter
}

func (c *Controller) HandleCommand(ctx context.Context, cmd *pb.DeploymentCommand) error {
    switch cmd.Action {
    case pb.DeploymentCommand_CREATE:
        return c.createDeployment(ctx, cmd.DeploymentId, cmd.Spec)
    case pb.DeploymentCommand_UPDATE:
        return c.updateDeployment(ctx, cmd.DeploymentId, cmd.Spec)
    case pb.DeploymentCommand_DELETE:
        return c.deleteDeployment(ctx, cmd.DeploymentId)
    case pb.DeploymentCommand_RESTART:
        return c.restartDeployment(ctx, cmd.DeploymentId)
    default:
        return fmt.Errorf("unknown action: %v", cmd.Action)
    }
}

func (c *Controller) createDeployment(ctx context.Context, id string, spec *pb.DeploymentSpec) error {
    // Update status: PROVISIONING
    c.statusReporter.Report(id, "PROVISIONING", "Creating infrastructure")
    
    // Phase 1: Terraform - Create AWS resources
    tfVars := c.buildTerraformVars(spec)
    tfOutput, err := c.terraformRunner.Apply(ctx, "deployment", id, tfVars)
    if err != nil {
        c.statusReporter.Report(id, "ERROR", fmt.Sprintf("Terraform failed: %v", err))
        return err
    }
    
    // Update status: CONFIGURING
    c.statusReporter.Report(id, "CONFIGURING", "Deploying Airflow components")
    
    // Phase 2: Helm - Deploy Airflow
    helmValues := c.buildHelmValues(spec, tfOutput)
    if err := c.helmManager.Install(ctx, spec.Namespace, "airflow", helmValues); err != nil {
        c.statusReporter.Report(id, "ERROR", fmt.Sprintf("Helm install failed: %v", err))
        return err
    }
    
    // Update status: STARTING
    c.statusReporter.Report(id, "STARTING", "Waiting for components to be ready")
    
    // Phase 3: Wait for healthy
    endpoints, err := c.healthChecker.WaitForHealthy(ctx, spec.Namespace, 5*time.Minute)
    if err != nil {
        c.statusReporter.Report(id, "ERROR", fmt.Sprintf("Health check failed: %v", err))
        return err
    }
    
    // Update status: RUNNING
    c.statusReporter.ReportWithEndpoints(id, "RUNNING", "Deployment is healthy", endpoints)
    
    return nil
}

func (c *Controller) buildTerraformVars(spec *pb.DeploymentSpec) map[string]interface{} {
    return map[string]interface{}{
        "deployment_name":    spec.Name,
        "namespace":          spec.Namespace,
        "db_instance_class":  c.getDBInstanceClass(spec.Resources),
        "redis_node_type":    c.getRedisNodeType(spec.Resources),
        "efs_throughput":     c.getEFSThroughput(spec.Resources),
    }
}

func (c *Controller) buildHelmValues(spec *pb.DeploymentSpec, tfOutput *terraform.Output) map[string]interface{} {
    return map[string]interface{}{
        "airflowVersion": spec.AirflowVersion,
        "executor":       spec.ExecutorType,
        "webserver": map[string]interface{}{
            "replicas":  1,
            "resources": c.getWebserverResources(spec.Resources),
        },
        "scheduler": map[string]interface{}{
            "replicas":  1,
            "resources": c.getSchedulerResources(spec.Resources),
        },
        "workers": map[string]interface{}{
            "replicas":  spec.Resources.MinWorkers,
            "resources": c.getWorkerResources(spec.Resources),
        },
        "postgresql": map[string]interface{}{
            "enabled": false, // Using external RDS
        },
        "externalDatabase": map[string]interface{}{
            "host":     tfOutput.Get("rds_endpoint"),
            "port":     5432,
            "database": "airflow",
            "user":     "airflow",
            "passwordSecret": spec.Namespace + "-db-credentials",
        },
        "redis": map[string]interface{}{
            "enabled": false, // Using external ElastiCache
        },
        "externalRedis": map[string]interface{}{
            "host": tfOutput.Get("redis_endpoint"),
            "port": 6379,
        },
        "dags": map[string]interface{}{
            "persistence": map[string]interface{}{
                "enabled":      true,
                "storageClass": "efs-sc",
                "size":         "10Gi",
            },
        },
    }
}
```

### 11.1 Multi-Cloud Provider Abstraction

```go
// pkg/providers/interface.go

package providers

import "context"

// CloudProvider defines the interface for cloud-specific operations
type CloudProvider interface {
    // Compute
    CreateKubernetesCluster(ctx context.Context, spec ClusterSpec) (*Cluster, error)
    DeleteKubernetesCluster(ctx context.Context, clusterID string) error
    
    // Database
    CreateDatabase(ctx context.Context, spec DatabaseSpec) (*Database, error)
    DeleteDatabase(ctx context.Context, databaseID string) error
    
    // Cache
    CreateCacheCluster(ctx context.Context, spec CacheSpec) (*CacheCluster, error)
    DeleteCacheCluster(ctx context.Context, cacheID string) error
    
    // Storage
    CreateFileSystem(ctx context.Context, spec FileSystemSpec) (*FileSystem, error)
    DeleteFileSystem(ctx context.Context, fsID string) error
    
    // Networking
    CreateVPC(ctx context.Context, spec VPCSpec) (*VPC, error)
    CreateSecurityGroup(ctx context.Context, spec SecurityGroupSpec) (*SecurityGroup, error)
    
    // Identity
    CreateServiceAccount(ctx context.Context, spec ServiceAccountSpec) (*ServiceAccount, error)
}

// ClusterSpec defines Kubernetes cluster requirements
type ClusterSpec struct {
    Name           string
    Version        string
    NodeGroups     []NodeGroupSpec
    VPCConfig      VPCConfig
    Tags           map[string]string
}

// DatabaseSpec defines managed database requirements
type DatabaseSpec struct {
    Name          string
    Engine        string // postgres, mysql
    Version       string
    InstanceClass string
    StorageGB     int
    MultiAZ       bool
    Encrypted     bool
}

// Provider factory
func NewProvider(cloud string, config ProviderConfig) (CloudProvider, error) {
    switch cloud {
    case "aws":
        return NewAWSProvider(config)
    case "gcp":
        return NewGCPProvider(config)  // Future
    case "azure":
        return NewAzureProvider(config) // Future
    default:
        return nil, fmt.Errorf("unsupported cloud provider: %s", cloud)
    }
}
```

### 11.2 Hybrid/Agent-Based Extension

The current architecture directly supports hybrid deployments:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    HYBRID DEPLOYMENT EXTENSION                               │

           │                                         │

           │                                         │ mTLS + JWT
           ▼                                         ▼
┌─────────────────────┐                   ┌─────────────────────┐
│   Data Plane        │                   │   Data Plane        │

└──────────┬──────────┘                   └──────────┬──────────┘    by customer
           │                                         │
           │ K8s API                                 │ K8s API

1. Data Plane Agent packaged as Helm chart for customer installation

5. We provide: Agent image, Terraform modules for AWS resources
```

### 12.3 Glossary

| Term | Definition |
|------|------------|
| **Control Plane** | Central management layer handling API, auth, billing, and orchestration |
| **Data Plane** | Infrastructure orchestration layer that manages Airflow deployments |
| **Runtime Plane** | Runtime environment where Airflow workloads execute |
| **Data Plane Agent** | Component in Data Plane that communicates with Control Plane |
| **Deployment** | A single Airflow environment (webserver, scheduler, workers, databases) |
| **Workspace** | Logical grouping of deployments within an organization |
| **Organization** | Top-level tenant with billing and user management |
